想知道Vue中路由的简单搭建以及应用吗？

发表于2022-05-07|更新于2022-05-18
|字数总计:935|阅读时长:3分钟|阅读量:

路由的简单搭建是必须要会的哦

基本知识了解

components 文件夹是经常放置的非路由组件,即全局组件
pages|views 文件夹是经常放置路由组件
下载路由插件

配置路由

项目中配置路由一般放置在 router 文件夹下

在 main.js 文件中需要对路由进行注册与引入

route 和 router 区别

1、路由组件一般放置在 pages|views 文件夹，费路由组件一般放置在 components 文件夹中
2、路由组件一般需要在 router 文件夹中进行注册(使用的是组件名字),非路由组件在使用时，一般都是以标签的形式使用
3、注册完路由，不管路由组件、还是费路由组件身上都有$route、$router 属性

$route:一般获取路由信息[路径、query、params等等]
$router:一般进行编程式导航进行路由跳转[push|replace]

路由的跳转

路由的跳转分为两种形式:声明式导航(router-link)以及编程式导航(push|replace)

声明式导航可以做的,编程式导航都可以做到;但是编程式导航除了可以进行路由跳转，还可以做其他的业务逻辑

声明式导航 router-link

router-link(务必要有 to 属性),可以实现路由的跳转

编程式导航 push|replace

利用组件实例的$route.push|replace 方法,可以实现路由的跳转

多次执行会抛出 NavigatinDuplicated 警告错误，最新的 vue-router 会引入 promise
通过给 push 方法传递相应的成功，失败的回调函数，可以捕获当前错误，但是治标不治本，将来在其他组件中使用 push|place，编程式导航依然有类似错误
想要直接解决此类错误，则需要重写 push|place

路由传递参数的方法

字符串形式

配置路由进行占位即可进行跳转

模板字符串形式

对象写法

必须在配置路由中命名路由,加上 name 属性

路由跳转传递参数时,对象写法可以是 name、path 形式，但需要注意的是,path 这种写法不能与 params 参数一起使用
